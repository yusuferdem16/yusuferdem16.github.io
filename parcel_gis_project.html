Project Overview

This project represents a complete full-stack web application designed for municipal government agencies to manage and analyze land parcels using geographic information systems. The system enables city planners and officials to:

  • View and query geographic data on interactive maps
  • Analyze parcel boundaries and properties
  • Filter and select parcels based on criteria (size, location, ownership, zoning)
  • Generate reports for administrative decision-making
  • Manage geospatial data with enterprise-grade database infrastructure

Tech Stack Highlights: C# .NET backend (66.1%), JavaScript/React frontend (21.8%), HTML/CSS (11.8%), with PostgreSQL+PostGIS for spatial data storage.

Spatial Query Example (C#)

Backend implements sophisticated PostGIS spatial queries using Entity Framework:

// DataAccess Layer: Spatial Query for Nearby Parcels
public async Task> GetNearbyParcels(
    double latitude, 
    double longitude, 
    int radiusMeters)
{
    var userPoint = new Point(
        new Coordinate(longitude, latitude)) 
        { SRID = 4326 };
    
    return await dbContext.Parcels
        .Where(p => p.Geometry.Distance(userPoint) 
            <= radiusMeters)
        .OrderBy(p => p.Geometry.Distance(userPoint))
        .Take(50)
        .ToListAsync();
}

Database: PostgreSQL + PostGIS

Data is persisted in PostgreSQL with PostGIS extension for geospatial operations.

Schema Design

The database schema stores vector geometries (polygons for parcel boundaries) with spatial indexes for fast queries:

-- Parcels table with geometry column
CREATE TABLE parcels (
    id SERIAL PRIMARY KEY,
    parcel_number VARCHAR(50) UNIQUE NOT NULL,
    area_sqm DECIMAL(12,2),
    zone_type VARCHAR(50),
    owner_id INT REFERENCES owners(id),
    geometry GEOMETRY(Polygon, 4326),
    created_at TIMESTAMP DEFAULT NOW()
);

-- Create spatial index for fast queries
CREATE INDEX idx_parcels_geometry 
    ON parcels USING GIST(geometry);

-- Example spatial query: Find parcels within 500m
SELECT id, parcel_number, area_sqm
FROM parcels
WHERE ST_DWithin(
    geometry, 
    ST_GeomFromText('POINT(32.8597 39.9334)', 4326),
    500
);

Key Features

  • Spatial Indexes: GIST (Generalized Search Tree) indexes enable
  • Geometry Storage: Polygons stored in WGS84 (SRID 4326) - standard GPS coordinate system
  • Scalability: PostgreSQL can efficiently handle 100k+ parcels with complex geometries

Deployment Flow

  1. Build Docker image: Compile C# backend into container
  2. Push to registry: Upload to Docker Hub or private registry
  3. Deploy to server: Run on Linux server in municipality
  4. Database migration: Apply schema migrations on PostgreSQL
  5. Enable HTTPS: SSL certificates for secure data transmission
