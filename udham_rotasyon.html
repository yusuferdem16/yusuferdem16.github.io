üå± Problem Statement

Turkey's major cities face two interconnected challenges mandated by sustainable transportation policies:

  • Traffic Congestion: High population density and economic activity create severe congestion in urban centers
  • Emissions Control: Vehicle emissions contribute significantly to air pollution and climate change
  • Limited Awareness: Drivers lack real-time information about congestion pricing zones and emission restrictions
  • Poor Transit Integration: Lack of seamless integration between private vehicles and public transportation options

Traditional approaches rely on penalties and restrictions. ROTASYON takes a different approach: guiding and incentivizing sustainable choices through intelligent routing and real-time information.

‚ú® Core Features & Architecture

1. üöó Vehicle Profile Registration System

Users register their vehicle details in a secure database for eligibility assessment against congestion pricing and low-emission zone (LEZ) regulations.

Data Captured:

  • Vehicle Type: Electric, hybrid, gasoline, diesel, etc.
  • License Plate: For zone entry validation
  • Engine Displacement: For emission classification
  • Emission Status: Euro standards (Euro 4, 5, 6, etc.)

Why This Matters: The system can instantly determine if a vehicle qualifies for zone access, enabling real-time decision-making in route optimization.

2. üó∫Ô∏è Intelligent Navigation & Route Management

The core innovation: dual-path routing based on vehicle eligibility.

Shortest/Fastest Route:
Vehicles that meet emission standards and congestion pricing requirements receive optimized routes that pass directly through restricted zones, minimizing travel time.

‚ùå For Ineligible Vehicles

Alternative Routes:
Bypass routes that avoid restricted zones, preventing fines and penalties. The system offers sustainable alternatives.

4. üìä Data Analytics & City Planning

üèóÔ∏è Technical Implementation

Technology Stack

Mobile App (iOS/Android) GPS/Mapping API Backend API PostgreSQL Database Cloud Infrastructure Real-time Analytics

System Architecture

Client Layer (Mobile App): Provides user interface, GPS tracking, and route display

API Layer (Backend): Handles route calculation, vehicle eligibility checks, pricing computation, and real-time notifications

Data Layer (Database): Stores vehicle profiles, zone boundaries, pricing rules, historical traffic data, and user journey logs

Key Algorithms

  • Vehicle Eligibility Filter: Real-time validation against current zone regulations
  • Multi-Objective Route Optimization: Balances shortest path, travel time, environmental impact, and user preferences
  • Congestion Prediction: ML-powered forecasting to suggest best departure times
  • Pricing Calculation: Dynamic congestion charges based on time-of-day and vehicle type
